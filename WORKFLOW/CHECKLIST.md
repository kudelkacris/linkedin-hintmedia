# CHECKLIST.md — Verificación Pre-Output

Ejecutar antes de mostrar cualquier diagnóstico o mensaje. Si algún ítem es NO → no mostrar. Corregir primero.

---

## CHECKLIST DE DIAGNÓSTICO

Antes de construir el diagnóstico:

- [ ] Leí el .md del prospecto completo
- [ ] Identifiqué el stage actual
- [ ] Leí el último mensaje del prospecto (no el último de Hint)
- [ ] Identifiqué qué información nueva apareció
- [ ] Clasifiqué el engagement (LOW / MEDIUM / HIGH)
- [ ] Identifiqué señales, objeciones y oportunidades
- [ ] Elegí UNA sola acción (MSG2 / SEG1 / SEG2 / WAIT / RECOVERY)
- [ ] Elegí UNA sola estrategia (CONSULTIVA / ENTRE_PARES / EXPLORATORIA)
- [ ] Justifiqué por qué esa persona dedicaría 20 minutos (si hay CTA de reunión)

---

## CHECKLIST DE MENSAJE

Antes de mostrar el mensaje al usuario:

- [ ] Respondí realmente el último mensaje del prospecto
- [ ] Incorporé información nueva — no repetí el ángulo anterior
- [ ] El primer sujeto NO es "nosotros" / "Hint" / "trabajo con"
- [ ] Hint aparece como consecuencia de la conversación, no como pitch
- [ ] El prospecto entiende qué es Hint antes de que llegue el dossier (si aplica)
- [ ] El dossier tiene contexto (qué contiene) antes del CTA (si aplica)
- [ ] Cada burbuja cumple un único objetivo
- [ ] Si elimino el párrafo de Hint, la conversación sigue teniendo sentido
- [ ] No usé ninguna frase de BLOCKLIST.md
- [ ] El mensaje conecta con al menos un win personal del prospecto (no solo el win de su empresa)
- [ ] El saludo es "Buenas [nombre]!" — nunca "Hola"
- [ ] No hay guiones largos (—)
- [ ] No hay signos de apertura (¡, ¿)
- [ ] No atribuí al prospecto algo que no dijo explícitamente

---

## PREGUNTA FINAL (obligatoria)

Si el prospecto leyera este mensaje, ¿sentiría que realmente lo escuché, o que estoy siguiendo un proceso?

Si la respuesta es "siguiendo un proceso" → reescribir completamente. No mostrar.
